# Стоматологія Коваля — сайт-лендінг

Готовий до публікації на GitHub Pages статичний сайт. Весь код (HTML/CSS/JS),
шрифти та ключові зображення (лого, фото лікаря, favicon) вбудовані в
`index.html` як base64 — сторінка працює повністю офлайн, без зовнішніх
запитів (окрім карти Google Maps і форми запису).

## Структура

```
index.html            — весь сайт (head + body + стилі + скрипти)
assets/img/
  og-preview.jpg       — картинка для прев'ю в соцмережах (Open Graph / Twitter Card)
  clinic-photo.jpg      — фото для мікророзмітки Schema.org (поле "image")
```

## Перед публікацією обов'язково перевірте

1. **Домен.** Мета-теги (`canonical`, `og:url`, `og:image`, JSON-LD) прописані
   під домен `https://koval-dental.com.ua/`. Якщо сайт публікується на
   GitHub Pages без підключеного власного домену (тобто адреса буде виду
   `https://username.github.io/repo/`), потрібно замінити ці посилання на
   реальну адресу — інакше прев'ю в соцмережах і Google не працюватимуть
   коректно.
2. **Форма запису → Telegram.** Токен бота та ID чату вписані прямо в
   JavaScript (це видно у вихідному коді сторінки — GitHub може позначити
   це попередженням про секрет при пуші, це очікувано). Технічно запит
   тепер надсилається так, щоб уникнути CORS-preflight блокування (яке,
   ймовірно, і було причиною того, що заявки не долітали раніше) — але
   **обов'язково протестуйте форму наживо** після публікації, оскільки
   з цього середовища розробки немає доступу до api.telegram.org, щоб
   перевірити доставку повідомлення напряму. Якщо після тесту заявки
   й далі не приходять — найнадійніше рішення це перенести відправку
   через невеликий backend/serverless-проксі (Cloudflare Worker,
   Netlify/Vercel function), який тримає токен у секреті на сервері.
3. **Google Maps.** Кнопка "Прокласти маршрут" і карта в блоці контактів
   використовують робоче посилання-пошук за адресою клініки. Поле
   `hasMap` у Schema.org веде на те саме — за бажання замініть на
   короткий фірмовий лінк `maps.app.goo.gl`, коли він буде під рукою.
